# Growth-Company-Prediction
📈 Growth Company Prediction & Review Automation

비재무 정보를 활용한 기술성장기업 예측 및 AI 심사 의견 자동화 시스템

1. 프로젝트 개요
1.1 프로젝트 배경

금융권의 기업·기술금융 평가에서는 여전히 재무제표가 핵심 자료로 활용되고 있습니다.
그러나 신설·초기 기술기업은 과거 실적이 부족하고 R&D 투자 비중이 높아, 재무정보만으로 미래 성장성을 판단하는 데 구조적 한계가 존재합니다.

특허·기술력·시장성 등 비재무 정보가 미래 성과를 설명하는 데 중요하다는 연구는 지속적으로 축적되고 있음에도,
실제 여신심사 현장에서는 여전히 정성 평가와 심사역의 경험에 크게 의존하고 있어 객관성·일관성·확장성 측면의 제약이 존재합니다.

최근 금융권은 AI·빅데이터를 활용한 기술신용평가 고도화를 시도하고 있으나,

기술평가 결과와 실제 경영성과의 연결

모델의 설명가능성(XAI) 확보

심사 의견의 표준화·자동화

는 여전히 해결되지 않은 과제로 남아 있습니다.

본 프로젝트는 이러한 문제의식에서 출발하여,
국내 제조업 구조를 대표하는 반도체·이차전지·디스플레이 소부장 기업을 대상으로
재무제표(정형 데이터)와 특허·뉴스(비정형 데이터)를 결합하고,
t-1년 정보로 t년 고성장 여부를 예측·설명하는 AI 기반 심사 지원 시스템을 구축하는 것을 목표로 합니다.

1.2 프로젝트 목표

반도체·이차전지·디스플레이 소부장 기업의 1년 후 성장 가능성 예측

롤링 패널 데이터 기반 모델 학습으로 일반화 성능 확보

정형 + 비정형 데이터 결합을 통한 예측력 향상 검증

SHAP 기반 설명 가능한 AI(XAI) + GenAI 심사 의견 자동 생성

금융기관 여신심사팀의 미래 우량 기술성장기업 선제 확보 의사결정 지원

1.3 프로젝트 요약
항목	내용

2. 데이터 설계
2.1 롤링 패널 데이터 구조 (핵심 설계)

단일 연도 기준 예측은 표본 수 부족으로 과적합 위험이 큽니다.
이에 본 프로젝트는 (기업, 연도) 단위의 롤링 패널 구조를 도입하여,

동일한 문제 정의
→ “t-1년 정보로 t년 성과 예측”

학습 샘플 수를 4~5배 확장

시간 축을 따른 일반화 성능 확보

를 동시에 달성하도록 설계했습니다.

예상 학습 샘플 규모

약 400~500개 기업 × 4개 연도
→ 1,600~2,000개 샘플

2.2 예측 시나리오와 실무 연계성

피처 시점: 연도 말 기준(t-1)

타겟 시점: 다음 해 3분기 누적 실적(t)

이는 실제 금융기관의
연간 기업 리뷰(Annual Review) → 대출 한도 설정 프로세스와 구조적으로 동일합니다.

2.3 데이터 누수 방지 원칙

모든 피처는 해당 연도 12월 31일 이전 정보만 사용

뉴스, 특허, 재무제표 모두 동일 기준 적용

타깃 연도 정보는 학습 과정에서 완전 배제

2.4 데이터 구성
정형 데이터 (DART OpenAPI)

재무제표: 매출, 영업이익, 자산, 부채, 자본

직원 현황: 종업원 수, 연구 인력

투자 지표: 현금흐름표 기반 CAPEX

비정형 데이터

특허: KIPRIS API (출원·요약·IPC·피인용)

기업 뉴스: 네이버 뉴스 API (메타·요약)

전방산업 뉴스: 공공데이터포털 뉴스 빅데이터

3. 타겟 변수 정의 (업종 중립화)
문제 인식

절대 성장률 기준은 업종·규모 편향 발생

“매출 +15%” 같은 기준은 실무적 방어가 어려움

해결 전략

KSIC 3자리 업종 내 상대 비교

분위수 기반 타겟 정의

타겟 정의

Primary 타겟
→ 동일 업종 내 전년 대비 매출 성장률 상위 30%

Secondary 타겟
→ Primary + 영업이익 흑자 + 부채비율 ≤ 200%

→ ‘성장성’과 ‘대출 적격성’을 분리 정의

4. 소부장 특화 피처 엔지니어링
주요 피처 그룹

시계열 성장 피처 (CAGR, 성장 가속도, 변동성)

CAPEX 투자 강도 및 추세

전방산업 뉴스 모멘텀

특허 활동량·질·기술 다양성

특허 텍스트 임베딩 (KoSBERT → PCA)

모든 임베딩·차원축소 과정은
훈련 데이터에서만 fit → 테스트는 transform만 적용하여 누수를 방지했습니다.
